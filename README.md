2. 남의 코드를 해석하는게 아니라 알고리즘(사고방식?)을 구글링 하는 것임
3. 이상한 자료구조 있으면, 해당 자료구조 관련된 문제들 중 쉬운거 먼저 풀어보고 이해하기
4. 시간이나 메모리가 엄청 차이날 때 (풀다가 이상하다 싶은거는 다시보기)
   이럴 때는 남의 풀이 뜯어보면서 어디를 과하게 했는지 
5. 아이패드에 실수한거 적어놓기 (앞으로 어떻게 풀어야할지)
